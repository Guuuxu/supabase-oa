# 文书签署管理页面改进

## 修改内容

### 1. 删除"签署记录"部分
删除了原有的单一签署记录列表展示，该列表只显示所有签署记录的基本信息。

### 2. 新增"文书签署状态"功能
参考文书管理下的"签署状态"页面，新增了分标签页展示的签署状态功能。

## 新功能特性

### 标签页结构
使用Tabs组件将签署记录分为两个标签页：
- **待签署**：显示状态为 `pending`、`employee_signed`、`company_signed` 的记录
- **已完成**：显示状态为 `completed`、`rejected`、`withdrawn` 的记录

每个标签页标题显示对应状态的记录数量。

### 待签署列表
**显示字段**：
- 公司名称
- 员工姓名
- 部门
- 类型（文书模板名称）
- 状态（带颜色徽章）
- 发送时间
- 签署时间

**操作按钮**：
- 查看详情（Eye图标）：查看签署记录详细信息
- 撤回签署（X图标）：仅对待签署状态显示，可撤回签署记录

### 已完成列表
**显示字段**：
- 公司名称
- 员工姓名
- 部门
- 类型（文书模板名称）
- 状态（带颜色徽章）
- 发送时间
- 签署时间

**操作按钮**：
- 查看详情（Eye图标）：查看签署记录详细信息
- 预览文件（FileText图标）：预览已签署文件（仅有文件时显示）
- 下载文件（Download图标）：下载已签署文件（仅有文件时显示）

### 状态徽章样式
使用 `getStatusBadgeVariant` 函数根据状态返回不同的徽章样式：
- `completed`（已完成）：default 样式（蓝色）
- `pending`（待签署）：secondary 样式（灰色）
- `employee_signed`、`company_signed`：outline 样式（边框）
- `rejected`（已拒绝）、`withdrawn`（已撤回）：destructive 样式（红色）

### 空状态提示
- 待签署列表为空时显示"暂无待签署记录"
- 已完成列表为空时显示"暂无已完成记录"
- 加载中显示骨架屏动画

## 用户体验改进

### 1. 清晰的状态分类
通过标签页将签署记录按状态分类，用户可以快速找到需要处理的待签署记录或查看已完成的记录。

### 2. 实时数量统计
标签页标题显示对应状态的记录数量，用户可以一目了然地了解待签署和已完成的记录数量。

### 3. 状态可视化
使用不同颜色的徽章区分不同签署状态，提高可读性。

### 4. 操作按钮优化
- 待签署列表：提供查看详情和撤回操作
- 已完成列表：提供查看详情、预览文件和下载文件操作
- 根据记录状态和文件存在情况动态显示操作按钮

### 5. 响应式布局
表格支持横向滚动，适配不同屏幕尺寸。

## 与"签署状态"页面的一致性

新增的"文书签署状态"功能与文书管理下的"签署状态"页面保持一致：
- 相同的标签页结构（待签署/已完成）
- 相同的字段展示
- 相同的状态徽章样式
- 相同的操作按钮布局
- 相同的空状态提示

## 技术实现

### 数据筛选
使用数组的 `filter` 方法根据状态筛选签署记录：
```typescript
// 待签署
signings.filter(s => s.status === 'pending' || s.status === 'employee_signed' || s.status === 'company_signed')

// 已完成
signings.filter(s => s.status === 'completed' || s.status === 'rejected' || s.status === 'withdrawn')
```

### 组件使用
- `Tabs`、`TabsList`、`TabsTrigger`、`TabsContent`：标签页组件
- `Card`、`CardHeader`、`CardTitle`、`CardContent`：卡片组件
- `Table`、`TableHeader`、`TableBody`、`TableRow`、`TableCell`、`TableHead`：表格组件
- `Badge`：徽章组件
- `Button`：按钮组件
- `Skeleton`：骨架屏组件

### 保留的功能
- 查看详情对话框
- 文件预览对话框
- 文件下载功能
- 撤回签署功能

## 注意事项
1. 删除了原有的"签署记录"单一列表展示
2. 新增的标签页结构更符合用户的使用习惯
3. 操作按钮根据记录状态和文件存在情况动态显示
4. 保留了所有原有的功能（查看详情、预览、下载、撤回）
