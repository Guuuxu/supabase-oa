# 九头鸟人事托管签署系统需求文档

## 1. 应用概述

### 1.1 应用名称
九头鸟人事托管签署系统

### 1.2 应用描述
面向多公司的电子签署系统,用于处理各类人力资源管理文书的电子签署,支持接入第三方电子签API接口。

### 1.3 参考架构图
使用用户提供的图片 screenshot.png 作为系统架构参考。

## 2. 用户与使用场景

### 2.1 目标用户
- 超级管理员:系统最高权限角色,负责整体系统配置与管理
- 主管:负责管理所属公司的文书发起、审批与签署流程
- 员工:接收并签署各类人力资源管理文书

### 2.2 核心使用场景
- 多公司人事文书的电子签署管理
- 员工入职、在职、离职全生命周期文书处理
- 薪酬工资条生成与电子签署
- 客户(公司)信息管理与服务状态跟踪

### 3.2 首页(看板)

**数据统计卡片区**
- 展示以下统计数据卡片:
  - 工资记录总数:系统中的工资记录总数
  - 待签署工资条:等待员工签署的工资条数量
  - 考勤记录总数:系统中的考勤记录总数
  - 待签署考勤表:等待员工签署的考勤确认表数量
  - 可根据业务需要扩展其他统计卡片

**快速操作区**
- 提供以下快速操作入口(共6个):
  - 员工管理:跳转至员工管理页面
  - 发起签署:跳转至发起签署页面
  - 文书模板:跳转至文书模板库页面
  - 公司管理:跳转至客户管理页面
  - 工资表管理:跳转至工资表管理页面
  - 薪酬签署管理:跳转至薪酬签署管理页面

### 3.3 客户管理

**客户列表页**
- 展示所有客户(公司)信息列表,包含字段:
  - 客户基本信息(公司名称、行业、地域、员工规模)
  - 客户创建时间
  - 客户服务起止时间
  - 客户服务状态
  - 客户到期数据
  - 客户签章数据
- 支持筛选方式:
  - 按公司名称筛选
  - 按到期时间筛选
  - 按行业筛选
  - 按地域筛选
  - 按员工规模筛选
- 支持新增客户操作

**客户详情页**
- 展示客户完整信息
- 支持编辑客户基本信息、服务起止时间、服务状态
- 展示该客户关联的员工列表
- 展示该客户的签章使用数据

### 3.4 员工管理

**员工列表页**
- 展示所有员工信息,包含字段:
  - 员工基本信息(姓名、证件类型、证件号码、性别、出生日期、联系电话)
  - 所属公司(公司名称,点击后跳转至该公司所有员工的考勤列表)
- 支持按公司筛选员工
- 支持新增员工操作
- 新增员工后自动弹出提示,询问是否立即办理入职手续(跳转至入职文书签署流程)

**员工详情页**
- 展示员工基本信息
- 展示员工当前状态及历史状态变更记录
- 展示员工关联的文书签署记录
- 展示员工劳动合同起止时间及签订次数(当前是第几次签订)
- 展示员工已签订的所有文书列表,支持按文书类型、签订年份筛选

**员工状态管理**
- 支持设置员工状态,可选值:在职、请假、休假、待岗、出差、离职
- 员工状态设为离职后,系统自动通过短信向员工手机号发送离职手续签署链接

### 3.5 文书管理

**文书模板库**
- 公司模板:每个公司独立维护自己的文书模板,模板关联至对应公司
- 通用模板库:所有公司、所有员工均可选择使用
- 模板按以下大类分类管理:

  - 入职管理:入职信息登记表、劳动合同、岗位职责、员工手册、规章制度、保密协议、竞业禁止协议、培训协议
  - 在职管理:劳动合同续签、劳动合同变更协议、岗位调整、薪酬调整、职级调整、限期返岗通知书、记过确认通知书、请假条
  - 离职管理:离职申请、离职交接确认表、解除劳动合同协议、离职证明、保密协议确认书、竞业协议确认书
  - 薪酬管理:考勤确认、绩效考核确认、工资条确认、考勤表
  - 证明开具:收入证明、离职证明、在职证明

- 支持新增文书模板

**发起签署页**
- 选择目标公司与目标员工
- 选择文书模板(支持跨模板多选)
- 同一大类中选择多个文书时,签署人仅需完成一次签署动作
- 每一大类文书支持全选或部分选择后一次性发起签署
- 签署流程规则:
  - 劳动合同类:员工签署 → 公司(用人单位)签署
  - 其他制度文书:员工单方签署
- 发起后通过短信或站内信通知相关签署人

**档案下载页**
- 支持查询和查看:
  - 文书签署进度
  - 签署状态(已签署、待签署、已拒签)
  - 签署人信息
  - 签署时间
  - 签署流程节点
- 支持按公司、员工、文书类型、签署状态等条件筛选

### 3.6 薪资管理

**工资结构模板页**
- 设置通用工资结构模板,所有公司均可使用
- 模板包含多个可编辑的工资项字段
- 支持添加、删除、修改工资项名称和字段代码
- 每个工资项可设置字段类型(如数字、文本等)

**工资表管理页**
- 支持上传公司每月工资表文件
- 系统根据关联的工资结构模板自动拆分为每位员工的独立工资条文件
- 工资条文件包含:员工姓名、部门、岗位、发放日期、各工资项明细、应发金额、扣款金额、实发金额等完整信息
- 拆分后的工资条支持独立电子签署
- 工资条生成后自动关联至薪酬签署模块,成为待签署文件

**薪酬签署管理页**
- 展示所有待签署及已签署的薪酬文件列表(工资条、考勤确认表、绩效考核表等)
- 支持按公司、员工、月份、签署状态筛选
- 点击立即签署按钮,系统向对应员工手机号发送电子签署链接短信
- 发薪日当日,系统自动向员工手机号批量发送薪酬电子签署链接
- 展示签署状态(待签署、已签署、已拒签)及签署时间
- **批量操作区域:**
  - 选中多个薪酬文件后显示批量撤回按钮
  - 选中多个薪酬文件后显示批量删除按钮

### 3.7 用户管理

**用户列表页**
- 展示系统内所有用户信息
- 支持新增、编辑、停用用户
- 支持为用户分配角色

**角色设置页**
- 支持自主创建角色
- 支持对角色权限进行增加和删除
- 可配置的权限项包括:
  - 客户管理:公司录入、客户信息查看、客户签章管理、客户到期提醒设置
  - 员工管理:员工录入、员工信息查看、员工状态管理、员工考勤记录查看
  - 文书管理:文书模板增加、文书发起、签署情况查看、签署数据调取
  - 薪资管理:工资结构模板设置、工资表上传、薪酬签署管理、工资条生成
  - 系统配置:第三方电子签API接口配置、身份认证配置、数据查询配置
  - 用户权限管理:用户列表管理、角色设置、权限分配
  - 看板:首页数据统计查看、快速操作入口使用
  - 下属管理:管理直属下属的文书发起与审批流程

### 3.8 系统配置

## 4. 业务规则与逻辑

### 4.1 文书模板归属规则
- 每个公司拥有独立的文书模板集合,模板与公司一一关联
- 通用模板库独立于公司模板,所有公司和员工均可访问和使用

### 4.2 批量签署规则
- 同一大类中选择多个文书一次性签署时,签署人仅需完成一次签署动作
- 每一大类文书支持全选或部分选择后批量发起

### 4.3 签署流程规则
- 劳动合同类文书:须按顺序完成员工签署 → 公司(用人单位)签署两个节点
- 其他制度文书:员工单方签署即完成

### 4.4 员工生命周期自动化规则
- 新增员工后:自动弹出提示,询问是否立即办理入职手续
- 劳动合同到期前30天:
  - 通过站内信提醒员工
  - 通过短信提醒公司联系人
  - 通过短信提醒即将到期的员工
- 第二次到期提醒时:同步提醒公司(用人单位)需要和员工签订无固定期限劳动合同
- 员工状态设为离职后:自动通过短信向员工手机号发送离职手续签署链接
- 发薪日当日:系统自动向员工手机号批量发送薪酬电子签署链接

### 4.5 工资条生成规则
- 上传工资表后,系统依据关联的工资结构模板字段进行匹配拆分
- 每位员工生成一份独立的完整工资条文件
- 工资条生成后自动进入薪酬签署管理模块的待签署列表

### 4.6 角色权限规则
- 超级管理员拥有所有权限,不受角色权限配置限制
- 其他角色的权限由超级管理员或具备角色分配权限的用户进行配置
- 用户可被分配一个或多个角色,权限取并集

### 4.7 薪酬签署批量操作规则
- 支持选中多个薪酬文件后进行批量撤回操作
- 支持选中多个薪酬文件后进行批量删除操作
- 批量操作需进行二次确认,确保操作安全

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 员工手机号为空时触发短信通知 | 系统提示手机号缺失,跳过该员工的短信发送,并在操作结果中标注发送失败原因 |
| 上传工资表字段与工资结构模板不匹配 | 系统提示字段匹配异常,要求用户确认或重新选择模板后再执行拆分 |
| 签署人拒签 | 签署状态更新为已拒签,通知文书发起人,支持重新发起签署 |
| 劳动合同到期提醒发送失败(短信) | 记录发送失败日志,支持手动重新发送 |
| 同一员工同一文书重复发起签署 | 系统提示该文书已存在待签署记录,由操作人确认是否继续发起 |
| 角色删除时仍有用户关联该角色 | 系统提示存在关联用户,禁止删除,需先解除用户与该角色的关联 |
| 客户服务到期未续签 | 客户服务状态自动更新为已到期,并在看板中标注 |
| 批量撤回时存在已签署文件 | 系统提示存在已签署文件无法撤回,仅对符合条件的文件执行操作 |
| 批量删除时存在已签署文件 | 系统提示存在已签署文件无法删除,仅对符合条件的文件执行操作 |

## 6. 验收标准

- 首页看板正确展示工资记录总数、待签署工资条、考勤记录总数、待签署考勤表等统计数据
- 首页快速操作区包含员工管理、发起签署、文书模板、公司管理、工资表管理、薪酬签署管理共6个入口,点击后正确跳转至对应页面
- 文书模板按5大类正确分类展示,公司模板与通用模板库相互独立
- 同一大类多文书批量签署时,签署人仅需完成一次签署动作
- 劳动合同签署流程严格按照员工签署 → 公司签署顺序执行
- 新增员工后自动弹出入职手续办理提示
- 劳动合同到期前30天,站内信与短信提醒均正常触发
- 第二次到期提醒时同步提醒公司需要签订无固定期限劳动合同
- 员工状态设为离职后,系统自动发送离职手续签署短信链接
- 发薪日当日,系统自动向员工批量发送薪酬电子签署链接
- 上传工资表后,系统正确拆分为每位员工的独立工资条,并自动进入薪酬签署管理待签署列表
- 薪酬签署管理页支持按公司、员工、月份、签署状态筛选,点击立即签署可向员工发送短信链接
- 档案下载页正确展示签署进度、状态、签署人信息、签署时间及流程节点
- 签署数据调取页支持统计、分析及数据导出
- 角色设置页支持自主创建角色并对权限进行增删操作,权限项配置完整
- 客户管理支持按公司名称、到期时间、行业、地域、员工规模5种方式筛选
- 第三方电子签API接口预留配置入口,身份认证功能(身份证二要素、人脸识别、企业三要素、人脸对比)配置项完整
- 员工详情页展示劳动合同起止时间、签订次数及已签订的所有文书记录,支持按员工、文书类型、签订年份筛选
- 员工列表页展示员工所属公司,公司名称为可点击链接,点击后进入该公司所有员工的考勤列表
- 薪酬签署管理页显示批量撤回和批量删除按钮,支持选中多个文件后执行批量操作
- 系统管理菜单内提供完整详细的使用手册,涵盖所有功能模块的操作说明
