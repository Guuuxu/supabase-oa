# 修复签署状态页面查看详情功能

## 问题描述
在文书签署状态页面点击"查看详情"按钮后，跳转的页面不正确。原因是：
1. 跳转路径为 `/signing-status/${id}`，但路由配置中没有对应的带ID参数的路由
2. 现有的 `/signing-status-detail` 路由对应的是签署统计页面，不是单个记录的详情页面

## 解决方案
创建新的签署记录详情页面 `SigningRecordDetailPage.tsx`，并配置正确的路由。

## 实现内容

### 1. 创建签署记录详情页面
文件：`src/pages/SigningRecordDetailPage.tsx`

**功能特性**：
- 使用 `useParams` 获取路由参数中的记录ID
- 调用 `getSigningRecord(id)` API 获取单条签署记录详情
- 展示完整的签署记录信息，包括：
  - 基本信息（公司名称、员工姓名、文书类型、创建时间）
  - 签署信息（员工签署状态和时间、公司签署状态和时间）
  - 员工信息（姓名、身份证号、联系电话、电子邮箱、部门）
  - 签署文件（已签署文件下载）
  - 备注信息

**页面布局**：
- 页面头部：返回按钮 + 标题 + 状态徽章
- 基本信息卡片：显示公司、员工、文书类型、创建时间
- 签署信息卡片：分别显示员工签署和公司签署的状态和时间
- 员工信息卡片：显示员工详细信息（如果有）
- 签署文件卡片：提供已签署文件下载（如果有）
- 备注信息卡片：显示备注内容（如果有）

**状态徽章**：
- 已完成（completed）：default 样式（蓝色）
- 待签署（pending）：secondary 样式（灰色）
- 员工已签署/公司已签署：outline 样式（边框）
- 已拒绝/已撤回：destructive 样式（红色）

**加载状态**：
- 显示骨架屏加载动画
- 加载失败时显示错误提示并提供返回按钮

### 2. 更新路由配置
文件：`src/routes.tsx`

**添加导入**：
```typescript
import SigningRecordDetailPage from './pages/SigningRecordDetailPage';
```

**添加路由**：
```typescript
{
  name: '签署记录详情',
  path: '/signing-status/:id',
  element: <SigningRecordDetailPage />,
  visible: false
}
```

**路由顺序**：
1. `/signing-status` - 签署状态列表页
2. `/signing-status/:id` - 单个签署记录详情页（新增）
3. `/signing-status-detail` - 签署统计页面（保持不变）

### 3. 页面功能
- **返回按钮**：点击返回到签署状态列表页
- **状态显示**：页面右上角显示当前签署状态的徽章
- **信息展示**：使用卡片布局展示各类信息，清晰易读
- **文件下载**：如果有已签署文件，提供下载按钮
- **响应式布局**：支持桌面和移动端显示

### 4. 错误处理
- 记录不存在时显示提示并自动返回列表页
- 加载失败时显示错误提示
- 提供友好的空状态显示

## 用户体验改进
1. **清晰的信息层级**：使用卡片和分组展示不同类型的信息
2. **图标辅助**：每个信息块都有对应的图标，提高可读性
3. **状态可视化**：使用不同颜色的徽章区分不同状态
4. **快速返回**：页面顶部提供返回按钮，方便用户导航
5. **文件下载**：已签署文件可以直接下载查看

## 技术实现
- 使用 React Router 的 `useParams` 获取路由参数
- 使用 `useNavigate` 实现页面跳转
- 使用 Lucide React 图标库提供图标
- 使用 shadcn/ui 组件库构建界面
- 使用 Sonner 提供 Toast 提示

## 注意事项
1. 路由配置中 `/signing-status/:id` 必须在 `/signing-status-detail` 之前，避免路由匹配错误
2. 详情页面设置 `visible: false`，不在侧边栏菜单中显示
3. 记录不存在时会自动跳转回列表页，避免用户停留在错误页面
