# 签署文件下载功能测试指南

## 功能说明

系统现在支持查看和下载每一份具体的签署文件（如劳动合同、员工手册、保密协议等）。

## 测试步骤

### 1. 登录系统
- 使用您的账号登录九头鸟人事托管签署系统

### 2. 进入签署数据调取页面
- 点击左侧菜单 "文书管理" → "签署数据调取"

### 3. 查看签署记录
- 在页面中可以看到签署记录列表
- 找到状态为 "已完成" 的记录（绿色Badge显示）

### 4. 查看签署文件列表
- 点击该记录右侧的 "查看文件" 按钮
- 会弹出一个Dialog，显示该签署记录包含的所有签署文件

### 5. 下载签署文件
- 在Dialog中可以看到文件列表，包括：
  - 文书名称（如：劳动合同、员工手册、保密协议）
  - 文件大小（如：12.95 KB）
  - 签署时间
- 点击任意文件右侧的 "下载" 按钮
- 文件会自动下载到您的浏览器默认下载目录

## 功能特点

1. **一对多关系**: 一个签署记录可以包含多份签署文件
2. **独立下载**: 每份文件都有独立的下载按钮
3. **文件信息**: 显示文件名称、大小、签署时间等元数据
4. **友好界面**: 使用Dialog展示文件列表，操作直观

## 注意事项

- 只有状态为 "已完成" 的签署记录才会显示 "查看文件" 按钮
- 如果签署记录没有关联的签署文件，Dialog中会显示 "暂无签署文件"
- 如果某个文件没有file_url，会显示 "无文件" 而不是下载按钮

## 技术说明

- 测试PDF文件来自W3C官方测试资源
- 实际生产环境中，文件应该存储在Supabase Storage中
- 下载文件名格式：`{文书名称}_{文件ID前8位}.pdf`

## 问题排查

如果无法下载文件，请检查：

1. ✅ 是否登录系统
2. ✅ 签署记录状态是否为 "已完成"
3. ✅ 是否点击了 "查看文件" 按钮
4. ✅ Dialog中是否显示了文件列表
5. ✅ 浏览器是否允许下载文件（检查浏览器下载设置）

## 后续开发

在实际使用时，需要：

1. 集成第三方电子签API，获取已签署的文件
2. 将签署完成的文件上传到Supabase Storage
3. 在签署完成时，调用`createSignedDocuments`函数创建签署文件记录
4. 将file_url设置为Supabase Storage中的实际URL
